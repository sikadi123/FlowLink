# 🌊 FlowLink 开发进度看板 (Checklist)

> **项目目标：** 基于 Netty + Spring Boot 的高性能即时通讯平台  
> **核心原则：** 核心先行、敏捷迭代、模块解耦

---

## 🛠 1. 基础架构与网关 (Infrastructure)

- [ ] **Netty 服务端骨架：** 搭建 WebSocket 服务，支持多客户端接入。
- [ ] **自定义通信协议：** 设计统一的 JSON/Protobuf 消息传输格式。
- [ ] **逻辑网关鉴权：** 在 WebSocket 握手阶段拦截请求，校验 JWT Token。
- [ ] **心跳检测机制：** 服务端自动清理超时未响应的“死连接”。
- [ ] **Session 管理：** 实现 UserId 与 Netty Channel 的双向绑定映射。

---

## 👤 2. 用户与社交体系 (User & Social)

- [ ] **账户系统：** 用户注册、登录、密码加密存储。
- [ ] **个人资料：** 修改昵称、上传/更换头像（集成本地存储或 MinIO）。
- [ ] **好友管理：** 
    - [ ] 搜索用户（精确匹配/模糊搜索）。
    - [ ] 发送好友申请（实时通知目标用户）。
    - [ ] 好友列表展示及在线/离线状态实时刷新。
 - [ ] **群组模块：** 
    - [ ] 创建群聊、邀请成员。
    - [ ] 查看群成员列表、退出群组。

---

## 💬 3. 核心通讯能力 (Messaging Core)

- [ ] **私聊 (P2P)：** 毫秒级文本消息转发。
- [ ] **群聊 (Group Chat)：** 基于群组 ID 的消息广播。
- [ ] **多媒体消息：** 
    - [ ] 表情包发送（字符映射或图片）。
    - [ ] 图片发送（先上传获取 URL，再推送消息）。
- [ ] **消息持久化：** 所有聊天记录异步存入 MySQL。
- [ ] **历史拉取：** 切换对话窗口时，自动分页加载历史聊天记录。

---

## 🧪 4. 稳定性与交互优化 (Experience)

- [ ] **离线消息处理：** 用户上线后，自动拉取离线期间收到的私聊消息。
- [ ] **自动重连逻辑：** 前端监听连接断开，实现指数退避自动重连。
- [ ] **已读/未读提示：** 简单标记未读消息数（基于前端内存或 Redis）。
- [ ] **消息确认 (ACK)：** 解决“消息到底发出去了没”的视觉反馈（显示发送中/发送成功）。

---

## 🌟 5. 挑战加分项 (Bonus - 视 4 周进度而定)

- [ ] **消息撤回：** 发送 2 分钟内允许撤回，同步更新所有在线客户端。
- [ ] **正在输入中：** 实时提示对方正在输入的状态。
- [ ] **敏感词过滤：** 简单的字符串匹配过滤违规内容。
- [ ] **分布式部署：** 使用 Redis Pub/Sub 实现跨服务器的消息转发。
- [ ] **压力测试报告：** 使用 JMeter 模拟 500+ 用户同时在线的性能指标。

---

## 📅 4 周时间线建议

1.  **Week 1 (基座):** 确定协议，打通 Netty 与 前端 WebSocket，实现“回声聊天”。
2.  **Week 2 (业务):** 完成登录、好友关系、单聊文字互发、消息入库。
3.  **Week 3 (扩展):** 群聊功能、图片发送、头像修改。
4.  **Week 4 (收尾):** 离线消息处理、UI 细节润色、压测、Bug 修复。
